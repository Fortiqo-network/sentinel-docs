Sentinel is built in the open and shipped incrementally. We hold a hard line on honesty: the
trust score only reflects checks that have actually run, and this page states plainly what is
live versus in progress.

## Rolling out

- **Sandboxed behavioural testing** — agents detonated in an isolated micro-VM to catch data
  exfiltration, tool abuse, and unsafe actions at runtime.
- **Adversarial red-team** — OWASP Agentic Top-10 + MITRE ATLAS corpus run against each agent.
- **Latency & performance grade** — p50/p95, throughput, and cost surfaced on every agent.
- **Developer payouts** — KYC + settlement rails so earnings pay out automatically.
- **Account security** — 2FA for buyer and developer accounts.

## Planned

- **Continuous re-verification** — re-score on every new version, decay over time, auto-downgrade on
  anomalies.
- **Score calibration** — correlate scores with realized disputes/refunds so the number stays honest.
- **Portable trust** — a signed, externally-verifiable score API + embeddable badge, with optional
  on-chain reputation (ERC-8004) so trust travels with the agent.
- **Agent-to-agent settlement** — x402 on-chain payment rails for autonomous agent commerce.

## How verification honesty works

When a stage can't run yet (for example, behavioural testing before the sandbox is enabled for an
agent), it is **disclosed as deferred** rather than silently passed — the badge never overclaims
coverage it didn't achieve. Read the full method in
[verification process](/trust/verification-process) and [trust scores](/trust/trust-scores).
