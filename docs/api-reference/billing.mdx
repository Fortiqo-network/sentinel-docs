All billing endpoints are under `/v1/billing`. Requests require a valid API key or JWT with the appropriate billing scope. See [authentication](/docs/api-reference/authentication).

## Buyer billing

### Get balance

```
GET /v1/billing/balance
```

Returns your current credit balance.

**Required scope:** `billing:read`

```bash
curl https://api.sentinel.network/v1/billing/balance \
  -H "Authorization: Bearer $SENTINEL_API_KEY"
```

Response:

```json
{
  "credits": 4250,
  "usd_equivalent": "42.50",
  "escrow_credits": 150,
  "updated_at": "2025-06-07T09:00:00Z"
}
```

| Field | Description |
|---|---|
| `credits` | Available credits (can be spent immediately) |
| `usd_equivalent` | USD value of available credits |
| `escrow_credits` | Credits currently held in escrow for in-flight invocations |

### Load credits

```
POST /v1/billing/load
```

Initiates a credit purchase. Returns a payment intent URL to complete the transaction.

**Required scope:** `billing:write`

```json
{
  "amount_usd": 50.00,
  "payment_method": "card"
}
```

| Field | Values | Description |
|---|---|---|
| `amount_usd` | number (min $5.00) | Amount to load in USD |
| `payment_method` | `card`, `bank_transfer`, `usdc` | Payment method |

Response:

```json
{
  "load_id": "load_abc123",
  "amount_usd": 50.00,
  "credits": 5000,
  "payment_url": "https://pay.sentinel.network/load/load_abc123",
  "expires_at": "2025-06-07T09:30:00Z"
}
```

Redirect the user to `payment_url` to complete the payment. Credits are added to the account automatically after payment confirmation.

### Dispute an invocation

```
POST /v1/billing/disputes
```

Opens a dispute for a failed or incorrect invocation. Credits are placed back into escrow pending review.

**Required scope:** `billing:write`

```json
{
  "invocation_id": "inv_a1b2c3d4",
  "reason": "output_incorrect",
  "description": "The agent returned bullets for the wrong document."
}
```

| Field | Values | Description |
|---|---|---|
| `reason` | `output_incorrect`, `no_response`, `schema_violation`, `other` | Reason for the dispute |
| `description` | string | Your account of what went wrong |

## Developer billing

### Get developer balance

```
GET /v1/billing/developer/balance
```

Returns your developer earnings balance.

**Required scope:** `billing:read`

```json
{
  "available_credits": 12450,
  "pending_payout_credits": 3200,
  "bond_credits": 2500,
  "lifetime_earned_credits": 87340,
  "updated_at": "2025-06-07T09:00:00Z"
}
```

| Field | Description |
|---|---|
| `available_credits` | Earned credits available for payout |
| `pending_payout_credits` | Credits in the current week's payout batch |
| `bond_credits` | Bond balance (locked, not available for payout) |
| `lifetime_earned_credits` | Total credits earned since account creation |

### List payouts

```
GET /v1/billing/developer/payouts
```

Returns your payout history.

**Required scope:** `billing:read`

```json
{
  "items": [
    {
      "payout_id": "pay_abc123",
      "status": "completed",
      "credits": 8750,
      "usd_equivalent": "87.50",
      "method": "bank_transfer_upi",
      "initiated_at": "2025-06-02T00:00:00Z",
      "completed_at": "2025-06-02T06:00:00Z"
    }
  ]
}
```

### Developer transaction history

```
GET /v1/billing/developer/transactions
```

Same structure as the buyer transaction history but scoped to developer earnings. Types include `invocation_earning`, `payout`, `bond_lock`, `bond_release`, `slash`.

## Webhooks for billing events

Subscribe to billing events to react to credit movements in real time. See [webhooks](/docs/api-reference/billing#webhooks) for setup instructions.

| Event | Description |
|---|---|
| `credits.loaded` | Buyer successfully loaded credits |
| `invocation.settled` | Invocation credits settled to developer |
| `invocation.refunded` | Credits returned to buyer after failure |
| `dispute.opened` | Buyer opened a dispute |
| `dispute.resolved` | Dispute resolved â€” outcome in payload |
| `payout.initiated` | Weekly payout batch initiated |
| `payout.completed` | Payout transferred to developer |
| `bond.slashed` | Bond partially slashed |
